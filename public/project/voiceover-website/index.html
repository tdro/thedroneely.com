Voiceover Website

Posted 7 May 2018 in projects; tagged with voiceover

https://www.edwinmattiacci.com

Voice acting is one of my interests, and I’ve made a website to manage just that. There were a few super simple html/css/javascript websites that I created before this one but I consider this my first formal website that actually does something.

It’s made using a combination of PHP for dynamic content and Hugo for static content – a MySQL database for the backend, and Nginx as the web server. Everything sits behind a front controller and the project structure is based on a very barebones implementation of the Laravel framework. To manipulate the content on the site, markdown files are edited and the static components of the website are recompiled by Hugo. The comment section is handled by a PHP commenting system called HashOver.

The original intention was to program a full CMS backend UI that would play nice with Hugo serverside to update the markdown files, but the web development world is either very clever or lazy? Recently I discovered the concept of a Headless CMS and yes — I honestly don’t have the slightest clue of what that term really means.

In the future, it would be nice to have the remaining CMS functionality integrated into a self-hosted Netlify instance (looks like it’s open source). This will use either gitlab’s or github’s API to update the static parts of the website and will provide not only version control but a stupid simple interface for making changes to the website. I’ve got some more reading to do.
